# Product Requirements Document: Gamification & Motivation System

## Executive Summary

### Product Vision
Transform Meridian into an engaging, motivating platform by adding game-like elements that encourage productivity, celebrate achievements, and foster healthy team competition—all while maintaining professional credibility.

### Business Objectives
1. Increase daily active users by 40% through engagement features
2. Improve user retention by 25% via habit formation (streaks)
3. Boost task completion rates by 30% through challenges and competition
4. Enhance team collaboration by 50% via social gamification
5. Create premium feature opportunity ($75K+ ARR potential)

### Target Users
- **Mike (Developer)**: Loves streaks, wants recognition for contributions
- **David (Team Lead)**: Wants to motivate team, celebrate wins
- **Sarah (Project Manager)**: Needs team engagement, progress visibility
- **Jennifer (Executive)**: Wants high-performing, motivated teams

## Core Features

### Feature 1: Achievement Badges 🏅
**Priority**: P0 (Critical)  
**User Story**: As a user, I want to unlock badges for accomplishments so I feel recognized and motivated to achieve more.

**Requirements**:

**Badge Categories** (50+ badges total):

1. **Task Achievements** (15 badges):
   - "First Steps" - Complete your first task
   - "Getting Started" - Complete 10 tasks
   - "Task Master" - Complete 100 tasks
   - "Marathon Runner" - Complete 1,000 tasks
   - "Speed Demon" - Complete 10 tasks in one day
   - "Night Owl" - Complete task after 10 PM
   - "Early Bird" - Complete task before 7 AM
   - "Deadline Dancer" - Complete 20 tasks on time
   - "Zero Inbox" - Have zero overdue tasks
   - "Priority Pro" - Complete 50 high-priority tasks

2. **Goal Achievements** (10 badges):
   - "Goal Setter" - Create first OKR
   - "Goal Achiever" - Complete first goal
   - "Key Results Champion" - Complete 50 key results
   - "OKR Master" - Complete 10 OKRs
   - "Consistent" - Update goals weekly for 4 weeks
   - "Ambitious" - Set 5 goals simultaneously
   - "Overachiever" - Complete goal 2 weeks early

3. **Team Achievements** (12 badges):
   - "Team Player" - Give 10 kudos
   - "Helpful Hero" - Receive 10 kudos
   - "Mentor" - Help 5 teammates
   - "Collaborator" - Work on 10 projects
   - "Communicator" - Send 100 messages
   - "Problem Solver" - Receive 5 problem-solving kudos

4. **Streak Achievements** (8 badges):
   - "Week Warrior" - 7-day streak
   - "Month Master" - 30-day streak
   - "Quarter Champion" - 90-day streak
   - "Century" - 100-day streak
   - "Legend" - 365-day streak

5. **Special Achievements** (10 badges):
   - "Early Adopter" - First 1000 users
   - "Innovator" - Create first automation
   - "Analyst" - View 100 reports
   - "Designer" - Customize workspace theme
   - "Completionist" - Unlock all common badges

**Badge Properties**:
- Name and description
- Icon/emoji representation
- Rarity (Common, Rare, Epic, Legendary)
- Point value (10-500 points)
- Unlock criteria (programmatic)
- Unlock date/timestamp
- Progress tracking (X/Y format)

**UI/UX**:
- Badge showcase page (grid view)
- Locked/unlocked states (grayscale vs color)
- Progress bars for in-progress badges
- Badge detail modal (how to unlock)
- Recent unlocks widget on dashboard
- Badge notifications (toast + modal)
- Share badge achievement (optional)

**Success Criteria**:
- 70% of users unlock at least 5 badges in first month
- 50% of users view badge showcase page
- Badge unlock notification engagement rate >80%

### Feature 2: Streak Tracker 🔥
**Priority**: P0 (Critical)  
**User Story**: As a user, I want to maintain activity streaks so I build consistent work habits and stay engaged.

**Requirements**:

**Streak Types** (5):
1. **Login Streak**: Consecutive days logging into Meridian
2. **Task Completion Streak**: Consecutive days completing at least 1 task
3. **Goal Update Streak**: Consecutive weeks updating goal progress
4. **Collaboration Streak**: Consecutive days helping teammates (kudos, comments)
5. **Learning Streak**: Consecutive days viewing help/documentation

**Streak Mechanics**:
- **Daily Reset**: Midnight in user's timezone
- **Grace Period**: 24-hour window to maintain streak
- **Streak Freeze**: 1 freeze per month (premium feature)
- **Streak Recovery**: Can recover once per month if broken
- **Milestone Rewards**: Extra points at 7, 30, 100, 365 days

**Streak Display**:
- Flame emoji with number (🔥 23 days)
- Progress bar to next milestone
- Longest streak badge
- Current vs longest streak comparison
- Streak calendar (heatmap view)

**Notifications**:
- Daily reminder if streak at risk (8 PM)
- Celebration when milestone reached
- Warning if about to break (11 PM)
- Recovery option when broken

**UI/UX**:
- Streak widget on dashboard
- Flame animation on streak extension
- Milestone celebration modal
- Heatmap calendar (GitHub-style)
- Streak comparison with friends (opt-in)

**Success Criteria**:
- 40% of users maintain 7+ day streak
- 20% of users reach 30-day streak
- 5% of users reach 100-day streak
- Streak reminder engagement >60%

### Feature 3: Leaderboard 🏆
**Priority**: P1 (High)  
**User Story**: As a team lead, I want to see who's most productive so I can recognize top performers and motivate the team.

**Requirements**:

**Leaderboard Types**:
1. **Task Completion** - Most tasks completed
2. **Goal Achievement** - Highest goal completion %
3. **Team Contribution** - Kudos + collaboration score
4. **Streak Champion** - Longest active streaks
5. **Quality Score** - On-time completion rate
6. **Overall Score** - Weighted combination of all

**Scoring Algorithm**:
```typescript
totalScore = (
  (tasksCompleted * 10) +
  (goalsCompleted * 100) +
  (keyResultsCompleted * 25) +
  (kudosReceived * 10) +
  (kudosGiven * 5) +
  (streakDays * 2) +
  (achievementsUnlocked * 50)
) * streakMultiplier
```

**Time Periods**:
- Today
- This Week
- This Month
- This Quarter
- All Time

**Scoping**:
- Workspace-wide
- Team-level
- Department-level
- Custom groups

**Privacy Controls**:
- **Opt-In Required**: Must explicitly join leaderboard
- **Anonymous Mode**: "User #123" instead of name
- **Friends Only**: Share rank only with connections
- **Public**: Visible to all workspace members
- **Opt-Out Anytime**: Can leave leaderboard

**Leaderboard UI**:
- Top 10 list with avatars
- Rank medals (🥇🥈🥉)
- Point breakdown on hover
- Rank change indicators (↑↓)
- User's rank highlighted
- Filter by category and period
- Podium visualization for top 3

**Features**:
- Real-time rank updates
- Rank history graph
- Achievement showcase for leaders
- "Catch up" suggestions for lower ranks
- Team aggregated score

**Success Criteria**:
- 60% opt-in rate
- 80% of participants check leaderboard daily
- 50% increase in task completion among participants
- Zero negative feedback about toxic competition

### Feature 4: Progress Rings 💍
**Priority**: P1 (High)  
**User Story**: As a user, I want Apple Watch-style progress rings so I can see my daily/weekly/monthly activity at a glance.

**Requirements**:

**Ring Structure** (3 concentric rings):

**Ring 1 (Outer - Blue)**: Daily Tasks
- Target: Complete X tasks today (user-defined, default 5)
- Progress: Tasks completed / Target
- Color gradient: Blue (#007AFF) → Light Blue (#5AC8FA)
- Completion: "Perfect Day" achievement

**Ring 2 (Middle - Green)**: Weekly Goals
- Target: Update Y goal key results this week (default 3)
- Progress: Key results updated / Target
- Color gradient: Green (#34C759) → Light Green (#30D158)
- Completion: "Productive Week" achievement

**Ring 3 (Inner - Red)**: Monthly Milestones
- Target: Complete Z milestones this month (default 2)
- Progress: Milestones completed / Target
- Color gradient: Red (#FF3B30) → Pink (#FF375F)
- Completion: "Milestone Master" achievement

**Ring Achievements**:
- Close all 3 rings: "Perfect Day" (+50 points)
- Close all rings 7 days: "Perfect Week" (+200 points)
- Close all rings 30 days: "Perfect Month" (+1000 points)

**UI/UX**:
- Large ring display (200px diameter)
- Animated fill on progress
- Sparkle effect on ring completion
- Numbers in center (tasks, goals, milestones)
- Click ring to see details
- Mini rings in dashboard widget (100px)
- Ring history calendar view

**Customization**:
- Set daily task target (1-20)
- Set weekly goal target (1-10)
- Set monthly milestone target (1-5)
- Choose ring colors (theme-based)
- Hide specific rings

**Success Criteria**:
- 50% of users close at least 1 ring daily
- 30% close all 3 rings weekly
- Ring widget viewed daily by 70% of users

**Collaboration Challenges**:

### Feature 6: Team Celebrations 🎉
**Priority**: P2 (Medium)  
**User Story**: As a team member, I want to celebrate wins with my team so we build camaraderie and acknowledge success.

**Requirements**:

**Celebration Triggers** (automatic):
1. **Project Completion**: When project marked as done
2. **Milestone Achievement**: When milestone reached
3. **Team Goal Complete**: When shared team goal completed
4. **Perfect Week**: When team has perfect week (all tasks done)
5. **Achievement Unlock**: When epic/legendary badge unlocked
6. **Streak Milestone**: When team member hits 30/100/365 day streak

**Celebration Types**:

1. **Confetti Explosion** 🎊
   - Multi-color confetti particles
   - Fall from top of screen
   - Physics-based animation
   - 3-second duration
   - Customizable colors (theme-based)

2. **Success Modal** 🌟
   - Full-screen takeover (dismissible)
   - Achievement/milestone displayed
   - Team avatars cascade in
   - Congratulations message
   - Share button (Slack, email, social)
   - Dismiss or view details

3. **Toast Celebration** 📣
   - Non-intrusive toast notification
   - Appears in top-right
   - Animated entry
   - Click to expand
   - Auto-dismiss after 5 seconds

4. **Fireworks** (premium) 🎆
   - Particle burst animation
   - Multiple launch points
   - Sound effects (optional)
   - Team celebration broadcast

**Team Broadcast**:
- When celebration triggered, notify all team members
- Show who achieved what
- Allow reactions (emoji)
- Optional message from achiever
- Team kudos suggestions

**Customization**:
- Enable/disable celebrations
- Choose celebration type
- Set celebration intensity (subtle, normal, epic)
- Mute sounds
- Opt-out of team broadcasts

**UI Components**:

**Success Criteria**:

## Technical Architecture

### Database Schema

```typescript
// Table 1: achievement_definitions
export const achievementDefinitions = pgTable("achievement_definitions", {
  id: text("id").primaryKey(),
  name: text("name").notNull(),
  description: text("description").notNull(),
  icon: text("icon").notNull(), // Emoji or icon name
  criteria: jsonb("criteria").notNull(), // { type: 'task_count', target: 100 }
  rarity: text("rarity").notNull(), // 'common', 'rare', 'epic', 'legendary'
  points: integer("points").notNull(),
  category: text("category").notNull(), // 'task', 'goal', 'team', 'streak', 'special'
  isActive: boolean("is_active").default(true),
  createdAt: timestamp("created_at").defaultNow(),
});

// Table 2: user_achievements
export const userAchievements = pgTable("user_achievements", {
  id: text("id").primaryKey(),
  userId: text("user_id").references(() => users.id, { onDelete: "cascade" }),
  achievementId: text("achievement_id").references(() => achievementDefinitions.id),
  progress: integer("progress").default(0), // Current progress toward achievement
  isUnlocked: boolean("is_unlocked").default(false),
  unlockedAt: timestamp("unlocked_at"),
  notified: boolean("notified").default(false),
  createdAt: timestamp("created_at").defaultNow(),
});

// Table 3: user_streaks
export const userStreaks = pgTable("user_streaks", {
  id: text("id").primaryKey(),
  userId: text("user_id").references(() => users.id, { onDelete: "cascade" }),
  streakType: text("streak_type").notNull(), // 'login', 'task', 'goal', 'collaboration', 'learning'
  currentStreak: integer("current_streak").default(0),
  longestStreak: integer("longest_streak").default(0),
  lastActivityDate: timestamp("last_activity_date"),
  streakStartDate: timestamp("streak_start_date"),
  freezesRemaining: integer("freezes_remaining").default(0), // Premium feature
  createdAt: timestamp("created_at").defaultNow(),
  updatedAt: timestamp("updated_at").defaultNow(),
});

// Table 4: leaderboard_scores
export const leaderboardScores = pgTable("leaderboard_scores", {
  id: text("id").primaryKey(),
  userId: text("user_id").references(() => users.id, { onDelete: "cascade" }),
  workspaceId: text("workspace_id").references(() => workspaces.id),
  scoreType: text("score_type").notNull(), // 'total', 'tasks', 'goals', 'kudos', 'quality'
  score: integer("score").notNull(),
  rank: integer("rank"),
  period: text("period").notNull(), // 'daily', 'weekly', 'monthly', 'all_time'
  isOptedIn: boolean("is_opted_in").default(false),
  isAnonymous: boolean("is_anonymous").default(false),
  calculatedAt: timestamp("calculated_at").defaultNow(),
});

// Table 5: daily_challenges
export const dailyChallenges = pgTable("daily_challenges", {
  id: text("id").primaryKey(),
  workspaceId: text("workspace_id").references(() => workspaces.id),
  challengeType: text("challenge_type").notNull(),
  title: text("title").notNull(),
  description: text("description").notNull(),
  difficulty: text("difficulty").notNull(), // 'easy', 'medium', 'hard', 'legendary'
  target: integer("target").notNull(),
  points: integer("points").notNull(),
  validDate: timestamp("valid_date").notNull(), // Challenge valid for this date
  isActive: boolean("is_active").default(true),
  createdAt: timestamp("created_at").defaultNow(),
});

// Table 6: user_challenge_progress
export const userChallengeProgress = pgTable("user_challenge_progress", {
  id: text("id").primaryKey(),
  userId: text("user_id").references(() => users.id, { onDelete: "cascade" }),
  challengeId: text("challenge_id").references(() => dailyChallenges.id),
  progress: integer("progress").default(0),
  target: integer("target").notNull(),
  isCompleted: boolean("is_completed").default(false),
  completedAt: timestamp("completed_at"),
  pointsEarned: integer("points_earned"),
  acceptedAt: timestamp("accepted_at").defaultNow(),
});

// Table 7: celebration_events
export const celebrationEvents = pgTable("celebration_events", {
  id: text("id").primaryKey(),
  workspaceId: text("workspace_id").references(() => workspaces.id),
  triggerUserId: text("trigger_user_id").references(() => users.id),
  celebrationType: text("celebration_type").notNull(), // 'confetti', 'fireworks', 'modal', 'toast'
  reason: text("reason").notNull(), // 'project_complete', 'achievement_unlock', etc.
  metadata: jsonb("metadata"),
  teamMemberIds: jsonb("team_member_ids"), // Who sees this celebration
  createdAt: timestamp("created_at").defaultNow(),
});

// Indexes for performance
CREATE INDEX idx_achievements_user ON user_achievements(user_id);
CREATE INDEX idx_achievements_unlocked ON user_achievements(is_unlocked);
CREATE INDEX idx_streaks_user_type ON user_streaks(user_id, streak_type);
CREATE INDEX idx_leaderboard_workspace_period ON leaderboard_scores(workspace_id, period);
CREATE INDEX idx_leaderboard_rank ON leaderboard_scores(rank);
CREATE INDEX idx_challenges_valid_date ON daily_challenges(valid_date);
CREATE INDEX idx_user_challenges ON user_challenge_progress(user_id, challenge_id);
```

## User Experience Flows

### Flow 1: Unlock First Achievement
1. User completes 10th task
2. System checks achievement criteria
3. "Getting Started" badge unlocked
4. Confetti animation plays (3 seconds)
5. Modal appears: "🎉 Achievement Unlocked!"
6. Badge displayed with description and points
7. Option to share on team feed
8. User clicks "Awesome!" to dismiss
9. Badge appears in user's profile

### Flow 2: Maintain Streak
1. User logs in for 7th consecutive day
2. Streak updates: 🔥 7 days
3. Toast notification: "7-day streak! Keep it up!"
4. Flame animation plays
5. "Week Warrior" badge unlocked
6. Points added to leaderboard (+14 points)

### Flow 3: Complete Daily Challenge
1. User sees daily challenges on dashboard
2. Challenge: "Complete 5 tasks today" (3/5)
3. User completes 4th task
4. Challenge updates to 4/5
5. User completes 5th task
6. Challenge completes! ✅
7. Celebration animation
8. Points awarded (+25)
9. Challenge marked complete

### Flow 4: Climb Leaderboard
1. User opts into workspace leaderboard
2. Initial rank: #47 (2,450 points)
3. Completes goal (+100 points)
4. Rank updates to #42 (↑ 5)
5. Green arrow shows rank improvement
6. Toast: "You moved up 5 spots!"
7. Next rank shown: #41 (need 50 more points)

### Flow 5: Close Progress Rings
1. User completes 5th task of the day
2. Blue ring closes (tasks ring complete)
3. Sparkle animation on ring
4. Toast: "Tasks ring closed! 🎯"
5. User updates goal progress
6. Green ring closes (goals ring complete)
7. User completes milestone
8. Red ring closes (all rings complete!)
9. "Perfect Day" modal with confetti
10. Badge unlocked + 50 bonus points
