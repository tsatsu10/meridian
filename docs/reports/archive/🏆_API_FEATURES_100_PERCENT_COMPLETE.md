## 💡 Key Technical Decisions

### 1. Argon2 for API Keys
**Decision**: Use Argon2 instead of bcrypt  
**Reason**: Memory-hard, GPU-resistant, modern  
**Impact**: Enhanced security against brute-force

### 2. HMAC-SHA256 for Webhooks
**Decision**: Sign all webhook payloads  
**Reason**: Prevent tampering, verify authenticity  
**Impact**: Enterprise-grade webhook security

### 3. Exponential Backoff for Retries
**Decision**: 1s, 5s, 15s retry delays  
**Reason**: Give external systems time to recover  
**Impact**: Higher delivery success rate

### 4. Scoped API Keys
**Decision**: Granular permission system  
**Reason**: Principle of least privilege  
**Impact**: Reduced security risk

### 5. Prefix-Based Key Lookup
**Decision**: Store key prefix for fast queries  
**Reason**: Faster validation without full scan  
**Impact**: Sub-millisecond key lookup
